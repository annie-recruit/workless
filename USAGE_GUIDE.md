# Workless, be sir - 사용 가이드 📖

## 빠른 시작

### 1. OpenAI API 키 설정

`.env.local` 파일에 API 키를 입력하세요:
```
OPENAI_API_KEY=sk-your-actual-api-key-here
```

### 2. 서버 실행 확인

서버가 실행 중입니다:
- 로컬: http://localhost:3000
- 네트워크: http://192.168.0.6:3000

브라우저에서 http://localhost:3000 을 열어주세요!

## 사용 방법

### 🧠 기억하기 탭

1. 텍스트 입력 창에 자유롭게 입력
   - 예: "채용 공고 작성 템플릿 아이디어 생각해봐야 함"
   - 예: "요즘 커리어 방향성 고민 중"
   - 예: "프로젝트 아이디어: 개인 비서 앱"

2. "기억해줘" 버튼 클릭

3. AI가 자동으로:
   - 주제 분류 (아이디어/업무/커리어 등)
   - 성격 파악 (단순기록/아이디어/고민 등)
   - 시간 성격 분석 (당장/언젠가/특정시점)
   - 관련 기억 연결
   - 맥락 묶음 생성

4. **조건부 제안**
   - 같은 주제를 3번 이상 입력하면
   - AI가 자동으로 다음 스텝을 제안합니다
   - 예: "이거 예전에 말한 채용 아이디어랑 묶어볼 수 있을 것 같아"

### 📚 보관함 탭

- 모든 기억을 맥락별로 묶어서 표시
- 클러스터 이름은 AI가 자동 생성
- 각 기억마다 자동 태그:
  - 주제 (파란색)
  - 성격 (보라색)  
  - 반복 횟수 (주황색, 2회 이상일 때)

### 💬 물어보기 탭

자연어로 질문하세요:

**예시 질문:**
- "요즘 내가 무슨 생각 많이 했어?"
- "아이디어 관련해서 뭐 쌓여 있어?"
- "업무 관련 기록 보여줘"
- "채용 관련 기록 정리해줘"
- "커리어 고민 요약해줘"

**응답 내용:**
1. 요약 - 관련 기억들을 짧게 정리
2. 관련 기억 목록 - 최대 5개 미리보기
3. 제안 (조건 충족 시) - 다음 스텝 아이디어

## 핵심 원칙

### ✅ 이런 점이 다릅니다

1. **선택 안 함**: 분류, 날짜, 카테고리 선택 ❌
2. **알아서 정리**: AI가 자동으로 분류하고 묶음
3. **맥락 중심**: 날짜순이 아닌 주제별 묶음
4. **조건부 제안**: 강요하지 않고, 필요할 때만
5. **자유 입력**: 형식 없음, 문장 엉망 OK

### ❌ 이런 건 아닙니다

- 할 일 관리 앱
- 일정 관리 앱
- 목표 추적 앱
- 습관 형성 앱

### ✨ 이런 용도로 좋습니다

- 떠오르는 아이디어 메모
- 고민거리 기록
- 나중에 써먹을 정보
- 반복되는 생각 추적
- 맥락별 정리가 필요한 기록

## 데이터 저장

- 모든 데이터는 로컬 SQLite 데이터베이스에 저장됩니다
- 파일 위치: `workless.db`
- 외부 서버에 업로드되지 않습니다 (OpenAI API 호출 제외)

## 팁

### 🎯 효과적인 사용법

1. **자주 입력하세요**
   - 짧게라도 자주 입력할수록 맥락이 쌓입니다
   - 완벽한 문장이 아니어도 OK

2. **반복되는 주제는 비슷한 단어로**
   - 예: 채용, 채용 아이디어, 채용 관련
   - AI가 자동으로 같은 클러스터로 묶습니다

3. **"물어보기"를 활용하세요**
   - 일주일에 한 번씩 "요즘 뭐 했어?" 물어보면
   - 자동으로 정리된 요약을 볼 수 있습니다

4. **제안은 참고만**
   - 조건부 제안은 강요가 아닙니다
   - 무시해도 됩니다!

### 🚀 고급 활용

1. **주간 회고**
   - 매주 금요일: "이번 주 뭐 생각했어?"
   
2. **아이디어 정리**
   - "아이디어 관련 기록 보여줘"
   - 누적된 아이디어를 한눈에

3. **고민 추적**
   - 같은 고민을 3번 입력하면
   - AI가 자동으로 패턴을 감지하고 제안

4. **맥락 재발견**
   - 보관함에서 클러스터별로 보면
   - 잊고 있던 생각들이 연결됩니다

## 문제 해결

### 서버가 시작되지 않는 경우

```bash
# 1. 패키지 재설치
npm install

# 2. 캐시 클리어
npm cache clean --force

# 3. node_modules 삭제 후 재설치
rm -rf node_modules package-lock.json
npm install
```

### API 키 오류

`.env.local` 파일이 프로젝트 루트에 있는지 확인:
```
C:\Users\user\workless-besir\.env.local
```

내용:
```
OPENAI_API_KEY=sk-your-actual-key-here
```

### 데이터베이스 초기화

```bash
# workless.db 파일 삭제 (모든 데이터 삭제됨)
rm workless.db
# 서버 재시작하면 자동으로 새 DB 생성
npm run dev
```
