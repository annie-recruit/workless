# Workless 실행 도와주기 강화 · 액션 초안 생성 — 정리 (Antigravity용)

> 아래 내용 그대로 복붙해서 사용하세요.

---

## 1. 방향성: Phase 1 — 실행 도와주기 강화 (지금 당장)

- **액션 아이템별 "초안 생성" 기능**
- **"다음 단계" 명확히 보여주기**
- **템플릿, 가이드 제공**
- **실행은 사용자가 직접**

---

## 2. 액션 초안 생성이란?

액션 플랜 카드에 단계별로 액션이 있으니까, **그 단계마다 액션 초안을 생성**하는 것.

- **"~조사해보기"** → 옆에 **관련 자료를 미리 서칭해서 링크** 띄워주기
- **"~~작성하기"** → **글 초안 생성**해서 보여주기  
- 실행은 사용자가 하고, Workless는 **재료만 준비**

---

## 3. 액션 타입별 초안 예시

| 액션 유형 | 예시 문구 | 초안 형태 |
|-----------|-----------|-----------|
| 조사형 | "경쟁 제품 비교 분석하기", "~조사해보기" | **참고 자료** 박스: 관련 링크 2~3개 (제목 + URL) |
| 글쓰기형 | "가이드 문서 초안 작성하기", "~~작성하기" | **글 초안** 박스: Markdown 초안 + 수정 버튼 + "이 초안 사용하기" |

---

## 4. UI 구성 (Pencil 디자인 기준)

- **액션 플랜 카드** 유지
- 각 액션 옆에 **✨ 초안 생성** 버튼 (주황 키컬러, 굵은 테두리)
- 클릭 시 **초안 섹션 확장**
  - **조사형**: 🔍 참고 자료 (노란 박스) + 링크 목록 + **복사** 버튼
  - **글쓰기형**: 📝 글 초안 (파란 박스) + 본문 + **수정** 버튼 + **🚀 이 초안 사용하기** 버튼
- 픽셀 디자인: 두꺼운 테두리, 주황/인디고, 픽셀 이모지

---

## 5. UX 플로우 요약

**조사형**  
`액션 선택 → [초안 생성] → 참고 자료 링크 표시 → 사용자가 링크 클릭/조사 → 체크 완료`

**글쓰기형**  
`액션 선택 → [초안 생성] → 글 초안 표시 → 사용자가 검토/수정 → [이 초안 사용하기] → 사용자가 직접 붙여넣기/실행 → 체크 완료`

---

## 6. 제품 정체성 (자동화 vs 생각앱)

- 처음 목표: **"생각앱"**, **"액션 아이템 도출"**, **"실행을 도와주는 앱"**, **"사고 구체화"**
- **자동 실행**을 넣으면 정체성이 흐려질 수 있음 → **Phase 1은 "실행 도와주기 강화"**로 가는 게 맞음
- **초안 생성 + 사용자 승인/실행** = 실행은 사용자가, Workless는 재료·가이드만 제공

---

## 7. 아키텍처 원칙 (나중에 자동화 넣을 때)

- **모든 실행은 큐 기반**으로 처리
- **Idempotency + 실행 로그 + 승인(기본 ON)** 포함
- 상태: `DRAFT → PROPOSED → APPROVED → RUNNING → SUCCEEDED/FAILED → (RETRYING/CANCELED)`

---

## 8. 지금 당장 할 것 (Phase 1)

1. 액션 플랜 카드 **단계별로 "초안 생성" 버튼** 추가
2. 액션 문구로 **조사형 vs 글쓰기형** 구분
3. **조사형**: 관련 링크 검색/수집 API + 참고 자료 UI
4. **글쓰기형**: AI 글 초안 생성 API + 글 초안 UI (수정, "이 초안 사용하기")
5. 실행은 전부 **사용자**가 (링크 클릭, 복붙, 발행 등)
